Markdownとは？

Markdownは、シンプルな記法で文書を装飾できる軽量マークアップ言語です。

HTMLを直接書くよりも圧倒的に楽で、多くのブログプラットフォームやObsidian、NotionなどのツールでサポートされIています。

「プレーンテキストで書いて、リッチなドキュメントを生成する」

これがMarkdownの哲学です。

基本の書式一覧

テキスト装飾

記法 表示 用途
**太字** 太字 重要な単語の強調
*斜体* 斜体 専門用語、外国語
~~打ち消し~~ 打ち消し 訂正、古い情報
`インラインコード` インラインコード コマンド、コード片
***太字斜体*** 太字斜体 特に強調したい場合

使い分けのポイント

正しい使い方:
- **太字**は1段落に1〜2箇所まで
- *斜体*は英語では強調、日本語ではあまり使わない
- `コード`は技術用語やファイル名に

悪い使い方:
- **この文章は**すべて**太字**にすると**読みにくい**

見出しとリスト

見出しの階層

# h1 - 記事タイトル（1つだけ）
## h2 - 大見出し
### h3 - 中見出し
#### h4 - 小見出し
##### h5 - 補足的な見出し
###### h6 - 最小の見出し

⚠️ h1は記事に1つだけ。本文ではh2から始めるのが基本です。

定義リスト

Markdown
: 軽量マークアップ言語の一つ。プレーンテキストで書式を指定できる。

HTML
: HyperText Markup Language。Webページの構造を定義する言語。

CSS
: Cascading Style Sheets。Webページの見た目を定義する言語。

書式を使いすぎない

最後に重要な注意点。

書式は調味料のようなもの

入れすぎると台無しになります

良い例

適度に太字を使い、必要な箇所だけを強調しています。

悪い例

この文章はすべての単語に太字を使っているので、どこが重要かわからなくなってしまいます。

まとめ

書式 主な用途
太字 重要な単語を強調
斜体 専門用語、外国語
コード コマンド、関数名
> 引用 他者の言葉、注目文
リスト 箇条書き、手順
テーブル 比較情報

書式は「読みやすさ」のためにあります。使いすぎに注意して、効果的に活用しましょう 📝
