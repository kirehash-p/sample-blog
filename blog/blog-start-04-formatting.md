## Markdownとは？

**Markdown**は、シンプルな記法で文書を装飾できる軽量マークアップ言語です。

HTMLを直接書くよりも圧倒的に楽で、多くのブログプラットフォームやObsidian、NotionなどのツールでサポートされIています。

> 「プレーンテキストで書いて、リッチなドキュメントを生成する」
> 
> これがMarkdownの哲学です。

## 基本の書式一覧

### テキスト装飾

| 記法 | 表示 | 用途 |
| --- | --- | --- |
| `**太字**` | **太字** | 重要な単語の強調 |
| `*斜体*` | *斜体* | 専門用語、外国語 |
| `~~打ち消し~~` | ~~打ち消し~~ | 訂正、古い情報 |
| `` `インラインコード` `` | `インラインコード` | コマンド、コード片 |
| `***太字斜体***` | ***太字斜体*** | 特に強調したい場合 |

### 使い分けのポイント

```markdown
正しい使い方:
- **太字**は1段落に1〜2箇所まで
- *斜体*は英語では強調、日本語ではあまり使わない
- `コード`は技術用語やファイル名に

悪い使い方:
- **この文章は**すべて**太字**にすると**読みにくい**
```

## 見出しとリスト

### 見出しの階層

```markdown
# h1 - 記事タイトル（1つだけ）
## h2 - 大見出し
### h3 - 中見出し
#### h4 - 小見出し
##### h5 - 補足的な見出し
###### h6 - 最小の見出し
```

> ⚠️ h1は記事に1つだけ。本文では**h2から始める**のが基本です。

### リストの種類

**箇条書き**:
- 項目A
- 項目B
  - ネストした項目
  - もう一つ
- 項目C

**番号付きリスト**:
1. 最初のステップ
2. 次のステップ
3. 最後のステップ

**チェックリスト**:
- [x] 完了したタスク
- [ ] 未完了のタスク
- [ ] もう一つのタスク

## 引用とコールアウト

### 基本の引用

> これは引用です。
> 複数行にまたがることもできます。

### ネスト引用

> 第一階層の引用
> 
> > 第二階層の引用
> > 
> > > 第三階層の引用

### 引用の活用例

引用は「他者の言葉」だけでなく、**注目してほしい文章**にも使えます。

> 💡 **Tip**: 引用の冒頭に絵文字を置くと、視覚的に目立ちます。

> ⚠️ **Warning**: 引用を使いすぎると、かえって読みにくくなります。

> 📝 **Note**: 補足情報を伝えるのにも効果的です。

## リンクと画像

### リンクの記法

```markdown
[表示テキスト](URL)
[Google](https://www.google.com)
```

結果: [Google](https://www.google.com)

### 内部リンク（Obsidian形式）

Obsidian形式のリンクは以下の4つのパターンがあります：

| パターン | 説明 |
| --- | --- |
| `[[article]]` | 基本形式 |
| `[[article\|text]]` | エイリアス付き |
| `[[article#heading]]` | 見出し指定 |
| `[[article#heading\|text]]` | 見出し+エイリアス |

例: [テーマの選び方の記事](/sample-blog/blog/blog-start-01-theme-selection.html)

見出し指定例: [前回のPREP法の解説](/sample-blog/blog/blog-start-03-article-structure.html#prep法最強の論理構成)

## テーブル（表）

### 基本構文

```markdown
| 列1 | 列2 | 列3 |
| --- | --- | --- |
| A1 | A2 | A3 |
| B1 | B2 | B3 |
```

### 配置の指定

```markdown
| 左寄せ | 中央寄せ | 右寄せ |
| :--- | :---: | ---: |
| Left | Center | Right |
```

| 左寄せ | 中央寄せ | 右寄せ |
| :--- | :---: | ---: |
| Left | Center | Right |
| データ | データ | 123 |

### テーブル作成のコツ

テーブルは**比較情報**に最適です。

| 無料ブログ | 有料ブログ | 自作ブログ |
| :---: | :---: | :---: |
| 簡単 | 普通 | 難しい |
| 無料 | 月額1000円〜 | サーバー代のみ |
| カスタマイズ× | カスタマイズ△ | カスタマイズ◎ |

## コードブロック

### インラインコード

文中で `const x = 1` のようにコードを示すときに使います。

### ブロックコード

言語名を指定すると、シンタックスハイライトが効きます。

```javascript
const greeting = (name) => {
  console.log(`Hello, ${name}!`);
};

greeting("World");
```

```python
def greeting(name):
    print(f"Hello, {name}!")

greeting("World")
```

```bash
# ファイルの一覧を表示
ls -la

# Gitの状態を確認
git status
```

コードブロックの詳しい使い方は [コードブロックの活用記事](/sample-blog/blog/blog-start-06-code-blocks.html#言語別のサンプル) で解説します。

## 数式（KaTeX）

### インライン数式

文中に $E = mc^2$ のように数式を埋め込めます。

ピタゴラスの定理: $a^2 + b^2 = c^2$

### ブロック数式

$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$

二次方程式の解の公式:

$$
x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}
$$

### 数式の活用シーン

- 技術ブログでの公式説明
- データサイエンス系の記事
- 教育コンテンツ

## 脚注

脚注は補足情報を本文の流れを妨げずに追加できます。[^1]

```markdown
本文中に脚注を入れます[^1]

[^1]: ここに脚注の内容を書きます
```

技術的な詳細や出典の明記に便利です。[^2]

[^1]: 脚注は記事の最後にまとめて表示されます。
[^2]: 出典: Markdown Guide (https://www.markdownguide.org/)

## 定義リスト

Markdown
: 軽量マークアップ言語の一つ。プレーンテキストで書式を指定できる。

HTML
: HyperText Markup Language。Webページの構造を定義する言語。

CSS
: Cascading Style Sheets。Webページの見た目を定義する言語。

## 書式を使いすぎない

最後に重要な注意点。

> **書式は調味料のようなもの**
> 
> 入れすぎると台無しになります

### 良い例

適度に**太字**を使い、必要な箇所だけを強調しています。

### 悪い例

**この文章は**すべての**単語に**太字を**使っている**ので、*どこが*重要か~~わからなく~~なって`しまいます`。

## まとめ

| 書式 | 主な用途 |
| --- | --- |
| **太字** | 重要な単語を強調 |
| *斜体* | 専門用語、外国語 |
| `コード` | コマンド、関数名 |
| > 引用 | 他者の言葉、注目文 |
| リスト | 箇条書き、手順 |
| テーブル | 比較情報 |

書式は「読みやすさ」のためにあります。使いすぎに注意して、効果的に活用しましょう 📝
